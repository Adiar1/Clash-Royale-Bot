How the bot works

Every command is a Discord slash command — type / and pick it from the list. A few conventions apply everywhere, so they're worth knowing once.

Tags or nicknames. Anywhere a clan tag is asked for, you can type the real tag (the # is optional and tags aren't case-sensitive) or a short server nickname set with /nicklink. Anywhere a player tag is asked for, you can instead @mention a Discord user and the bot uses their linked account.

Permissions. Commands marked Privileged are limited to roles you choose with /editperms. Until you set any privileged roles, everyone can use every command — so locking things down is opt-in.

Per-server settings. Anything marked Server data — nicknames, reminders, roles, recruiting — is stored for that Discord server only. Account links (your Clash tags) are global to your Discord account and follow you everywhere.

Interactive results. Tables from war and member commands come with dropdowns to sort and reorder columns, and a Download CSV button for the full data. These are marked Interactive.

/info

Shows a summary of the bot's features and a full command list right inside Discord — handy when you don't have this page open.

Clan Wars

War performance

Track fame earned and decks used, war by war. New members are flagged, and members who left but still fought are kept in the totals.

/currentwar <clan> Interactive

How the clan's current members are doing in the war that's on right now.

ParameterDetails
clanrequired — clan tag or a server nickname.
  • Sort by fame, name, decks, or tag, and reorder the columns from the dropdowns.
  • Download CSV exports the full table for spreadsheets.
/lastwar <clan> Interactive

The same table for the most recently finished war.

ParameterDetails
clanrequired — clan tag or a server nickname.
/nthwar <clan> <n> Interactive

War stats from a specific number of weeks ago — the bot remembers up to 10 wars.

ParameterDetails
clanrequired — clan tag or a server nickname.
nrequired — how many wars ago, 1–10 (1 is last war). You can also change this from a dropdown after it posts.
/stats <player> <from_war> <to_war>

A fame-history graph for one player over a range of wars, with average, median, high/low, a trend line, and a prediction for the next war.

ParameterDetails
playerrequired — player tag or an @mention of a linked user.
from_warrequired — the older end of the range, 1–10 weeks ago.
to_warrequired — the newer end, 1–10. Must be ≤ from_war (it's the more recent war).
Roster & profiles

Players & clans

Look up individual players, browse a full roster, and get data-backed suggestions for who to promote or kick.

/player <player>

A detailed profile: trophies, card levels, ranked results, war-deck stats, and clan role.

ParameterDetails
playerrequired — player tag or an @mention of a linked user.
/members <clan> Interactive

The current roster with names, tags, and clan ranks.

ParameterDetails
clanrequired — clan tag or a server nickname.
  • Toggle the view between all members, only new members, only former members, or everyone-but-new.
  • Sort and reorder columns, or download the CSV.
/clan <clan> Interactive

Every member with how many weeks ago they joined and their average fame — a quick read on tenure and consistency.

ParameterDetails
clanrequired — clan tag or a server nickname.
  • Linked members are shown as @mentions. Average fame looks back up to 10 weeks.
/viewlinks <clan>

Lists the clan's members, showing the linked Discord account for anyone who has one (and the in-game name for those who don't).

ParameterDetails
clanrequired — clan tag or a server nickname.
/rankings <tournament> Interactive

Tournament standings — every entrant's name, score, and rank.

ParameterDetails
tournamentrequired — the tournament tag.

How scoring works. The two commands below rank members by a blend of fame (99.2%), fame trend over time (0.5%), and commitment — weeks in the clan (0.3%). Brand-new members aren't scored. Tap More Info on Scoring on the result, or pick any member for their full fame graph.

/whotokick <clan> [n] [exclude_leadership] Interactive

Suggests the lowest-scoring members as kick candidates — a starting point, not a verdict.

ParameterDetails
clanrequired — clan tag or a server nickname.
noptional — how many to list, 1–24 (default 5).
exclude_leadershipoptional — set True to skip Co-Leaders and Leaders (default False).
/whotopromote <clan> [n] [exclude_leadership] Interactive

The flip side — the highest-scoring members who may have earned a promotion.

ParameterDetails
clanrequired — clan tag or a server nickname.
noptional — how many to list, 1–24 (default 5).
exclude_leadershipoptional — set True to skip existing leadership (default False).
Your accounts

Account linking

Link your Clash Royale accounts to your Discord profile once, and every command that takes a player can use an @mention instead of a tag. Links are global — they work in any server.

/link [user] Interactive

Opens the link manager — a panel with buttons to set your main tag, add alt accounts, attach DeckAI IDs, and remove accounts.

ParameterDetails
useroptional — manage someone else's links instead of your own. Managing another user requires a privileged role.
  • Your first account is the main; you can add up to 20 accounts total.
  • A DeckAI ID is optional and only needed for /spy_ai.
/profile [user] Interactive

Shows the Clash accounts linked to you (or someone you mention), with clan and trophies. Pick any account to open its full player profile.

ParameterDetails
useroptional — view another user's linked accounts.
/wipelinks [user] Privileged Interactive

Removes specific linked tags from an account via a select menu — useful for cleaning up someone else's stale links.

ParameterDetails
useroptional — whose links to remove (defaults to you).
Admin & configuration

Server setup

One-time configuration for your Discord server: nicknames, permissions, position roles, and automated war-day reminders.

/nicklink <clan> [nickname] Privileged Server data

Gives a clan a short nickname you can type instead of its tag in any clan command.

ParameterDetails
clanrequired — the clan tag to nickname.
nicknameoptional — under 5 characters. Leave it empty to delete the existing nickname.
/viewnicks Server data

Lists every clan nickname set in this server, alphabetically. Anyone can use it.

/editperms Privileged Server data Interactive

Chooses which roles count as “privileged.” Selecting a role toggles it in or out of the set.

Until you add at least one privileged role, all privileged commands are open to everyone. Set this up early.

/viewperms Server data

Shows the current privileged roles. Anyone can use it.

/editmemberroles Privileged Server data Interactive

Maps Discord roles to the in-game positions Member, Elder, and Co-Leader.

/viewmemberroles Server data

Shows which Discord role is mapped to each position. Anyone can use it.

/reminders <clan> Privileged Server data Interactive

Sets up automatic reminders that ping members who still have attacks left, on war days (Thursday–Sunday). Run it again to edit or delete.

ParameterDetails
clanrequired — clan tag or a server nickname.

Setup walkthrough

  1. Pick a channel

    Where the reminders get posted. The bot needs View Channel and Send Messages there.

  2. Pick a timezone

    Only used to show the reminder times in your local clock — the schedule itself is anchored to the war day.

  3. Pick one or more times

    Choose hours within the war day (each runs 10:00 → 10:00 UTC). Reminders then fire automatically at those times on war days.

  • Running /reminders on an already-configured clan opens an edit menu: send one now, change times, change channel, or delete.
Clan-family recruiting

Recruiting, on autopilot

Keeps a live “recruits needed” number for every clan with almost no work for leaders. The bot reads each clan's member count and figures the open slots itself — leaders are only pinged when a clan actually loses someone, and they answer with one tap.

One-time setup (privileged)

  1. Set a recruiting channel — /setrecruitchannel #channel

    The bot opens one thread per clan here. It needs View Channel, Send Messages, Create Public Threads, and Send Messages in Threads in that channel (the command checks for you).

  2. Assign a manager per clan — /setclanmanager <clan> @leader

    The person in charge of that clan. Assign more than one to share it; run it again on the same person to unassign.

That's the whole setup. From here the number stays current on its own; the commands below are only for checking in or overriding.

What a leader sees

When a clan loses a member, the bot posts in that clan's thread and @-mentions the manager(s):

👋 1 member left Verdant — is the number below right?
Recruits needed: 3
Members 47/50 · 3 open · 🔄 auto-tracked
Use suggested +1 −1 Set exact… We're full

Use suggested keeps the clan on autopilot. The other buttons pin a number the bot won't overwrite — though it'll still check in if the clan loses more members. When someone joins, the number drops on its own with no ping. Buttons keep working even after the bot restarts.

/setrecruitchannel <channel> Privileged Server data

Picks the channel where the per-clan recruiting threads live. One-time.

ParameterDetails
channelrequired — a text channel the bot can post and create threads in.
/setclanmanager <clan> <user> Privileged Server data

Assigns — or unassigns, it toggles — who manages a clan's recruiting.

ParameterDetails
clanrequired — clan tag or a server nickname.
userrequired — the member to assign or unassign. Assign several to share a clan.
/viewclanmanagers Privileged Server data

Lists who manages recruiting for each clan, and warns if the recruiting channel isn't set yet.

/editneeds <clan> <number> Privileged Server data

Pins a recruits-needed number by hand — handy for “I'm about to kick three, list us for three more.”

ParameterDetails
clanrequired — clan tag or a server nickname.
numberrequired — recruits needed, 0–50. 0 clears the clan from the list.
  • A pinned number isn't auto-overwritten. Tap Use suggested on a prompt to hand it back to auto-tracking.
/viewneeds Privileged Server data

The whole family's needs at a glance — every clan that needs recruits, most-needed first, with current member counts. This is the recruiter's dashboard.

Scouting & stats

Deck-level scouting for war duels, powered by DeckAI.

/spy_ai <opponent> [someone_else] Interactive

Scouts an opponent's available war-duel decks, shows your win rate against each, and — as the duel plays out — suggests your best remaining deck for the next match.

ParameterDetails
opponentrequired — the player tag of who you're dueling.
someone_elseoptional — @mention the person actually dueling, if it isn't you.
  • Requires a DeckAI ID on the dueling player's account — add one in /link.
  • After each match, tell the bot which decks were played and it recommends your next pick.

Looking for the fame-history graph? That's /stats over in the War section.

Good to know

Tips & legend

Tags are forgiving. The # is optional and capitalization doesn't matter. If you keep typing the same tag, give it a nickname with /nicklink.

@mentions stand in for tags. On player commands, mention a linked user instead of hunting down their tag.

Every table exports. War, member, clan, and ranking tables all have a Download CSV button for deeper analysis.

Markers. A “new member” badge means they joined after the last war ended; a “former member” marker means they've since left but still logged war activity.

Stuck? Run /info in Discord for a built-in cheatsheet, or contact the developer @adiar.
